# Java Question Set Cast

Shared character and business names for Java scenario-based questions. Use sparingly — one name + one site per scenario, max. Flavor should never make a stem longer than its plain-phrased version.

## Characters

| Name | Role | Typical use |
|---|---|---|
| **Aria Thorne** | Senior Java developer at Silverkeep Systems | Default protagonist. Core syntax, debugging, method behavior, general code review. |
| **Brom Ironfoot** | Junior developer / intern | The learner. Compile errors, beginner misconceptions, tracing program flow. |
| **Lyra Vex** | QA automation engineer | Unit tests, assertions, edge cases, bug reproduction. |
| **Captain Doran Kell** | Staff engineer / architect | OOP design, interfaces, inheritance, tradeoff and design-decision scenarios. |
| **Mira Sollen** | Product engineer / support liaison | User-reported defects, null handling, validation, exception behavior. |
| **Old Hask** | Build and release engineer | `javac`, JARs, classpath, packages, imports, Maven/Gradle, CI-flavored setup issues. |
| **Zinna Arc** | Backend / platform engineer | Collections, streams, APIs, data processing, service-style code examples. |

## Businesses / Sites

| Name | Type | Typical use |
|---|---|---|
| **Silverkeep Systems** | Mid-size software company | Default application-development context. |
| **Ember Forge Labs** | Startup feature team | Rapid iteration, refactoring, prototype code, changing requirements. |
| **The Gilded Tankard POS** | Small business desktop app | GUI, event handling, file I/O, practical line-of-business examples. |
| **Thornwood Clinic Records** | Regulated business application | Validation, exception handling, auditing, careful data processing. |
| **Marketstone Exchange Engine** | High-throughput backend service | Performance, threading, queues, efficient collections usage. |
| **Ravenshade Academy Portal** | School management application | Intro-level CRUD, classes/objects, forms, multi-module beginner examples. |

## Conventions

- No plotlines — each question stands alone.
- Match character to topic so names start to cue the domain (Lyra → testing, Hask → build tooling, Doran → design).
- Brom is the "wrong diagnosis to correct" character.
- Keep the software context light; the Java concept should remain the focus.
- Prefer software-team framing over epic fantasy action. The theme should color the setting, not change the task.
